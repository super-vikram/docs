## Wallet Requirements

SuperVaults uses [Dynamic.xyz](https://dynamic.xyz) for wallet connection and JWT issuance. You need:

- An **EVM-compatible wallet** (MetaMask, Rabby, Frame, or any WalletConnect-compatible wallet)
- ETH for gas on each supported chain (vault creation, on-chain operations)
- The wallet address must be assigned a **manager role** on at least one vault before accessing curator features

<Note>
View-only users are added off-chain (no wallet required for that tier). However, performing any vault operation — including ladder changes — requires at minimum a secondary manager role.
</Note>

## API Base URLs

Each backend service has its own base URL. Use these when building automated tooling:

| Service | Base URL |
|---------|----------|
| Erebor (Go management API) | `https://erebor.superform.xyz` |
| Superman (Python legacy) | `https://superman-api.superform.xyz` |
| Strategy Engine (Go) | `https://strategy.superform.xyz` |
| OMS (Go DeFiX) | `https://oms.superform.xyz` |

<Note>
These base URLs reflect the production environment. The Curator App env variables map to these via `NEXT_PUBLIC_EREBOR_API_URL`, `NEXT_PUBLIC_SUPERMAN_API_URL`, `NEXT_PUBLIC_STRATEGY_API_URL`. Confirm current staging URLs with the Superform team before building automation scripts.
</Note>

## Getting Your First Role

If you are onboarding to an existing vault, the primary manager must:
1. Navigate to **User Management** in the Curator App
2. Add you as a secondary manager (requires on-chain tx) or view-only user (off-chain)

If you are creating a new vault, you automatically become the primary manager at creation.
