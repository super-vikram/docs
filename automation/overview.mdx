## Why Automate?

Manual vault management is limited by human response time. Automated curators can:

- Monitor vault state 24/7 and respond to market changes in seconds
- Rebalance deposit ladders based on real-time APY rankings
- Pause services instantly when anomalies are detected
- Execute consistent rule-based strategies without emotional bias

SuperVaults supports full programmatic control via its four API backends.

## Authentication for Automation

The Curator App uses Dynamic.xyz for browser-based wallet auth. For automation scripts, you need to generate a JWT without the browser:

1. Sign an EIP-712 auth message with your wallet's private key
2. Submit the signature to Dynamic.xyz to receive a JWT
3. Use the JWT as `Authorization: Bearer <jwt>` on all requests

Once you have a JWT, verify it against Erebor:

<CodeGroup>

```python Python
import requests

EREBOR_URL = "https://erebor.superform.xyz"
JWT = "<your-jwt>"

headers = {"Authorization": f"Bearer {JWT}"}

# Confirm auth and get vault roles
resp = requests.get(f"{EREBOR_URL}/api/v1/auth/me", headers=headers)
resp.raise_for_status()
user = resp.json()

print(f"Authenticated as: {user['wallet_address']}")
print(f"Vaults: {[v['vault_address'] for v in user['vaults']]}")
```

```typescript TypeScript

console.log(`Vaults: ${user.vaults.map((v: any) => v.vault_address)}`);
```

## Safety Principles

<Warning>
Automated scripts have full manager-level access. A bug can modify live vault configuration. Follow these principles before deploying any automation.
</Warning>

### Test on Base First

Base (chain ID: 8453) has lower gas costs and faster confirmation times. Run all automation scripts against Base vaults before Ethereum mainnet.

### Rate Limit Your Polling

| Endpoint | Recommended Poll Interval |
|----------|--------------------------|
| Vault state | 15 seconds |
| Analytics overview | 60 seconds |
| Service health | 30 seconds |
| Ladder state | On-demand (after changes) |

### Implement Circuit Breakers

Stop automation if consecutive API errors are detected:

```python
MAX_CONSECUTIVE_ERRORS = 5
consecutive_errors = 0

def safe_api_call(fn):
    global consecutive_errors
    try:
        result = fn()
        consecutive_errors = 0
        return result
    except Exception as e:
        consecutive_errors += 1
        if consecutive_errors >= MAX_CONSECUTIVE_ERRORS:
            alert_and_stop(f"Circuit breaker: {consecutive_errors} consecutive errors")
        raise
```

### Never Modify Production Without Validation

Before pushing ladder changes:
1. Fetch current state
2. Validate proposed change is meaningful (e.g., order actually changed)
3. Check that yield sources in new order are all still whitelisted
4. Apply change
5. Re-fetch and verify change was applied
