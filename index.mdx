## Overview

SuperVaults is a tokenized vault infrastructure built on top of the Superform ecosystem. Curators deploy and manage ERC-4626 vaults that automatically allocate depositor funds across multiple yield sources — protocol vaults, Pendle Principal Tokens, lending markets, and more.

<Info>
SuperVaults is live on **Ethereum (chain ID 1)** and **Base (chain ID 8453)** with $130M+ in deposits across the broader Superform ecosystem and 140,000+ users.
</Info>

## Architecture

SuperVaults is composed of four backend services coordinating around on-chain contracts:

```
Curator UI (superman.superform.xyz)
       │
       ├──► Erebor API (Go)           – Vault management, auth, analytics, merkle, registry
       ├──► Superman Python (legacy)  – Ladder config, execution history, keeper health
       ├──► Strategy Engine (Go)      – Automated rule-based strategies, DeFiX intent dispatch
       └──► OMS (Go)                  – DeFiX v0.1 intent execution, on-chain settlement
                 │
       ┌─────────┴──────────┐
  On-chain Contracts        Indexers / Subgraph
  (SuperVault, Strategy,    (historical data,
   Escrow, Oracle)          events)
```

### Four Backends

| Service | Language | Role |
|---------|----------|------|
| **Erebor** | Go | Primary management API. Vault CRUD, auth, analytics (16 endpoints), merkle tree management, registry, audit logs. |
| **Superman** | Python | Legacy operations backend. Keeper ladder config, execution history, Celery worker health. |
| **Strategy Engine** | Go | Rule-based strategy automation. Translates curator-defined rules into DeFiX intents. |
| **OMS** | Go | DeFiX v0.1 protocol implementation. Receives intents from Strategy Engine, executes on-chain. |

### On-Chain Components

| Contract | Purpose |
|----------|---------|
| **SuperVault** | ERC-4626 tokenized vault. Holds depositor assets, delegates allocation. |
| **SuperVaultStrategy** | Manages yield source whitelist (merkle root), keeper authorization. |
| **Escrow** | Holds pending cross-chain redemption requests (ERC-7540). |
| **Oracle** | Price-per-share calculation and staleness tracking. |

## Key Standards

**ERC-4626** — The vault tokenization standard. Every SuperVault is ERC-4626 compliant, meaning deposits mint vault shares and redemptions burn them at the current price-per-share (PPS).

**ERC-7540** — Async deposit/withdraw for cross-chain flows. Co-authored by Vik Arun and 0xTimePunk. Enables Superform's cross-chain deposit infrastructure to queue requests and fulfill them asynchronously.

## What Curators Do

Curators are the operators of SuperVaults. As a curator you:

- Deploy vaults and configure underlying assets
- Whitelist yield sources and assign price oracles
- Configure ladder ordering (deposit priority, redemption priority, reserves)
- Manage merkle tree authorization for keeper hooks
- Automate rebalancing via the Strategy Engine
- Monitor vault health and execution history
- Manage access for secondary managers and view-only users
