## About Superman

Superman is a Python (Celery + Flask/FastAPI) service powering the operational layer of SuperVaults. It manages:

- **Ladder configuration** — deposit/redemption/reserve/claim/swap priorities
- **Execution history** — deposit and redemption execution logs
- **Service health** — 7 Celery worker statuses and infrastructure health

**Base URL:** `https://superman-api.superform.xyz`  
**Auth:** `Authorization: Bearer <jwt>` (same JWT as Erebor)  
**OpenAPI spec:** ❌ Does not yet exist

## Ladder Endpoints

### Get All Ladders

```
GET /ladder/ladders
Authorization: Bearer <jwt>
```

Returns all ladder configurations. The client filters by `vault_address` and `chain_id` client-side.

```json
[
  {
    "vault_address": "0x...",
    "chain_id": 8453,
    "deposit_priority": [...],
    "redemption_priority": [...],
    "reserve_percent": 5,
    "is_enabled": true
  }
]
```

### Get Vault State

```
GET /vaults/{normalizedAddress}
Authorization: Bearer <jwt>
```

Returns vault-level state (enabled status, etc.).

### Update Deposit Priority

```
PUT /ladder/{chainId}/{vaultAddress}/deposit-priority
Authorization: Bearer <jwt>
Content-Type: application/json

{
  "deposit_priority": [
    { "address": "0xYieldSource1..." },
    { "address": "0xYieldSource2..." }
  ]
}
```

### Update Redemption Priority

```
PUT /ladder/{chainId}/{vaultAddress}/redemption-priority

{
  "redemption_priority": [
    { "address": "0xYieldSource1...", "min_balance_to_keep": "1000000000000000000" }
  ]
}
```

`min_balance_to_keep` is in wei.

### Update Reserve

```
PUT /ladder/{chainId}/{vaultAddress}/reserve
{ "reserve_percent": 5 }
```

Range: 0–100 (integer percentage).

### Toggle Vault Enabled

```
PUT /vaults/{chainId}/{vaultAddress}/enabled
{ "enabled": true }
```

Master switch for all keeper workers for this vault.

### Claim Rewards Ladder

```
GET /ladder/{chainId}/{vaultAddress}/claim-rewards-ladder
PUT /ladder/{chainId}/{vaultAddress}/claim-rewards-ladder

{
  "claim_rewards_ladder": [
    {
      "token_address": "0x...",
      "floor_amount": "1000000000000000000",
      "is_enabled": true
    }
  ]
}
```

### Claimable Rewards (Live Balances)

```
GET /ladder/{chainId}/{vaultAddress}/claimable-rewards
```

Returns current accumulated reward token balances.

### Swap Ladder

```
GET /ladder/{chainId}/{vaultAddress}/swap-ladder
PUT /ladder/{chainId}/{vaultAddress}/swap-ladder

{
  "swap_ladder": [
    {
      "token_address": "0x...",
      "floor_amount": "500000000",
      "is_enabled": true
    }
  ]
}
```

### Swappable Tokens (Live Balances)

```
GET /ladder/{chainId}/{vaultAddress}/swappable-tokens
```

## Service Health Endpoints

### Infrastructure Health

```
GET /health
Authorization: Bearer <jwt>
```

```json
{
  "components": [
    { "component": "api", "status": "operational", "last_checked": "...", "message": null },
    { "component": "database", "status": "operational", ... },
    { "component": "redis", "status": "operational", ... },
    { "component": "celery", "status": "operational", ... },
    { "component": "subgraph", "status": "operational", ... },
    { "component": "rpc", "status": "operational", ... }
  ]
}
```

### Service Status (All 7 Workers)

```
GET /services/status
Authorization: Bearer <jwt>
```

```json
{
  "services": [
    {
      "name": "fulfill_deposits",
      "display_name": "Fulfill Deposits",
      "status": "healthy",
      "last_run": "2025-03-01T10:00:00Z",
      "next_run": "2025-03-01T10:01:00Z",
      "schedule_seconds": 60,
      "success_rate": 0.98,
      "avg_execution_time": 4.2,
      "total_executions": 8400,
      "is_enabled": true
    }
  ]
}
```

### Pause a Service

```
POST /services/{service_name}/pause
Authorization: Bearer <jwt>
```

### Resume a Service

```
POST /services/{service_name}/resume
Authorization: Bearer <jwt>
```

Valid service names: `fulfill_deposits`, `fulfill_redeems`, `fulfill_claims`, `fulfill_swaps`, `redemption_queue_engine`, `cleanup_processed_redeems`, `tx_manager`

### Update Schedule

```
PUT /services/schedules/{service_name}
Authorization: Bearer <jwt>

{ "interval_seconds": 60 }
```

### Scheduler Status

```
GET /services/scheduler/status
Authorization: Bearer <jwt>
```

```json
{
  "last_sync_at": "2025-03-01T10:00:00Z",
  "next_sync_in_seconds": 30,
  "sync_interval_seconds": 60,
  "loaded_schedules": ["fulfill_deposits", ...],
  "is_leader": true,
  "instance_id": "worker-1",
  "status": "healthy"
}
```

## Error Codes

| Code | Meaning |
|------|---------|
| `401` | JWT missing or expired |
| `403` | Not a SuperVaults Manager — wallet has no manager role |
| `404` | Vault not found |
| `422` | Validation error in request body |
