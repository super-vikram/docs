## Overview

The Ladder page controls how Superman's keeper workers allocate and deallocate vault assets. Five tabs cover every aspect of automated capital management.

<Info>
Ladder configuration is stored in and served from the **Superman Python** backend, not Erebor. All ladder endpoints use `apiClient` pointing to `NEXT_PUBLIC_SUPERMAN_API_URL`.
</Info>

## Vault Enabled Toggle

At the top of the Ladder page is the **Vault Enabled** master toggle. When disabled:
- All keeper workers stop processing for this vault
- Deposits and redemptions queue but are not fulfilled
- No yield source interactions occur

Use this for planned maintenance. For emergency stops, use the [Pause](/ui/pause) page.

## Tab 1: Deposit Priority

Controls which yield sources receive free vault assets first.

- The ordered list shows each yield source with its name, address, protocol, and current allocation
- **Drag to reorder** — top of list = deployed first
- Workers process the list top-to-bottom until free assets are exhausted or all sources are full

**API:** `PUT /ladder/{chainId}/{vaultAddress}/deposit-priority`

```json
// Request body
{
  "deposit_priority": [
    { "address": "0xYieldSource1..." },
    { "address": "0xYieldSource2..." }
  ]
}
```

## Tab 2: Redemption Priority

Controls which yield sources are unwound first to fulfill redemptions.

- Same drag-and-drop interface as Deposit Priority
- Each item also has a `min_balance_to_keep` field — workers will not withdraw below this threshold
- `min_balance_to_keep` protects against full liquidation of illiquid positions

**API:** `PUT /ladder/{chainId}/{vaultAddress}/redemption-priority`

```json
{
  "redemption_priority": [
    {
      "address": "0xYieldSource1...",
      "min_balance_to_keep": "1000000000000000000"
    }
  ]
}
```

<Tip>
Set `min_balance_to_keep` to a non-zero value for yield sources with exit fees or liquidity constraints. This prevents workers from triggering expensive partial redemptions.
</Tip>

## Tab 3: Reserve

Sets the percentage of vault assets held in the vault (not deployed to yield sources).

- Range: 0–100%
- Default recommendation: 5–10% for typical DeFi vaults
- Higher reserve = faster redemptions but lower yield
- Workers will not deploy assets beyond `(1 - reserve_percent)` of TVL

**API:** `PUT /ladder/{chainId}/{vaultAddress}/reserve`

```json
{ "reserve_percent": 5 }
```

## Tab 4: Claim Rewards

Configures the `fulfill_claims` worker — when to claim accumulated reward tokens from external protocols (e.g., Merkl distributions, Aave incentives).

For each reward token:
- **Floor Amount** — minimum accumulated balance before claiming. Below this threshold, claiming is skipped.
- **Is Enabled** — toggle per token
- **Claimable Amount** — live balance queryable via `GET /ladder/{chainId}/{vaultAddress}/claimable-rewards`

**APIs:**
- `GET /ladder/{chainId}/{vaultAddress}/claim-rewards-ladder` — view current config
- `PUT /ladder/{chainId}/{vaultAddress}/claim-rewards-ladder` — update thresholds
- `GET /ladder/{chainId}/{vaultAddress}/claimable-rewards` — live balances

## Tab 5: Swap

Configures the `fulfill_swaps` worker — when to swap accumulated non-underlying tokens back to the vault's underlying asset.

For each swappable token:
- **Floor Amount** — minimum token balance before swapping
- **Is Enabled** — toggle per token
- **Token Balance** — live balance from `GET /ladder/{chainId}/{vaultAddress}/swappable-tokens`

**APIs:**
- `GET /ladder/{chainId}/{vaultAddress}/swap-ladder` — view current config
- `PUT /ladder/{chainId}/{vaultAddress}/swap-ladder` — update thresholds
- `GET /ladder/{chainId}/{vaultAddress}/swappable-tokens` — live balances

## Optimistic UI

Ladder updates use optimistic UI — the UI reflects your change immediately while the API call is in flight. On API error, the change is rolled back and an error toast is shown.

## All Ladder API Endpoints

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/ladder/ladders` | GET | All ladder configs (filtered client-side) |
| `/vaults/{normalizedAddress}` | GET | Vault state |
| `/ladder/{chainId}/{vault}/deposit-priority` | PUT | Update deposit order |
| `/ladder/{chainId}/{vault}/redemption-priority` | PUT | Update redemption order |
| `/ladder/{chainId}/{vault}/reserve` | PUT | Set reserve % |
| `/vaults/{chainId}/{vault}/enabled` | PUT | Master keeper toggle |
| `/ladder/{chainId}/{vault}/claim-rewards-ladder` | GET/PUT | Claim thresholds |
| `/ladder/{chainId}/{vault}/claimable-rewards` | GET | Live reward balances |
| `/ladder/{chainId}/{vault}/swap-ladder` | GET/PUT | Swap thresholds |
| `/ladder/{chainId}/{vault}/swappable-tokens` | GET | Live token balances |

<Warning>
Amounts in all ladder APIs are in **wei** (18-decimal fixed-point by default, adjusts per token decimals). Format using `parseToWei`/`formatFromWei` utilities or equivalent in your language.
</Warning>
