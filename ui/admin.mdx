## Overview

The Admin page provides data backfill utilities for Erebor's historical analytics. Use these when vault data is missing from the analytics database — for example, after onboarding a new vault that has existing on-chain history, or after a data pipeline incident.

<Info>
Backfill operations are long-running (up to 120s per request). The API client timeout is overridden to 120s for admin endpoints. Require `registry_maintainer` permission.
</Info>

## Backfill Types

### APY Backfill

Recomputes and stores APY snapshots for a vault over a specified date range.

```bash
POST /api/v1/admin/backfill/apy
{
  "chain_id": 8453,
  "vault_address": "0x...",
  "days": 30
}
```

**Max days:** 90

Response:
```json
{
  "snapshots_updated": 28,
  "snapshots_failed": 2,
  "snapshots_skipped": 0,
  "errors": ["2025-01-03: missing PPS data for this date"]
}
```

### TVL Backfill

Recomputes and stores TVL snapshots for a vault.

```bash
POST /api/v1/admin/backfill/tvl
{
  "chain_id": 8453,
  "vault_address": "0x...",
  "days": 30
}
```

**Max days:** 90

Response:
```json
{
  "days_backfilled": 29,
  "days_failed": 1,
  "days_requested": 30,
  "errors": ["2025-01-15: RPC error fetching block"]
}
```

### Events Backfill

Re-indexes deposit, withdrawal, and other on-chain events for a vault.

```bash
POST /api/v1/admin/backfill/events
{
  "chain_id": 8453,
  "vault_address": "0x...",
  "days": 90
}
```

**Max days:** 365 (events go further back than APY/TVL)

Response:
```json
{
  "deposits_found": 142,
  "withdrawals_found": 38,
  "events_created": 178,
  "events_skipped_duplicate": 2
}
```

## Using the Backfill UI

1. Select **Backfill Type** (APY, TVL, or Events)
2. Select **Chain** from the dropdown
3. Select **Vault** from the dropdown
4. Choose **Days** — presets: 7, 30, 90 (Events adds 365)
5. Click **Run Backfill**
6. Wait for results (up to 120s — a loading indicator shows progress)
7. Review the results summary (updated/failed/skipped counts)

<Warning>
Backfills can temporarily spike database load. Run during off-peak hours when possible. Do not run multiple concurrent backfills for the same vault.
</Warning>

## API Reference

| Endpoint | Method | Timeout | Description |
|----------|--------|---------|-------------|
| `/api/v1/admin/backfill/apy` | POST | 120s | APY snapshots (max 90 days) |
| `/api/v1/admin/backfill/tvl` | POST | 120s | TVL snapshots (max 90 days) |
| `/api/v1/admin/backfill/events` | POST | 120s | On-chain events (max 365 days) |
