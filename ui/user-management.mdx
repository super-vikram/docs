## Overview

The User Management page controls who has access to your vault. Three tiers are managed here:

- **Primary Manager** — read-only display (set at vault creation, immutable)
- **Secondary Managers** — on-chain delegates; add/remove via wallet transactions
- **View Only** — off-chain read access; add/remove without gas

<Info>
The full manager list is fetched from `GET /api/v1/users?vault_address={}&chain_id={}&limit=100`. Returns all roles grouped by user.
</Info>

## Tab 1: Primary Manager

Shows the primary manager's wallet address and display name. This is the on-chain vault owner set at creation.

- **Cannot be changed** without governance-level contract interaction
- All other management actions (user management, yield sources, etc.) can be delegated to secondary managers
- Protect this key — it is the root of trust for the vault

## Tab 2: Secondary Managers

Secondary managers are on-chain delegates. Multiple secondary managers are supported.

### Adding a Secondary Manager

1. Click **Add Secondary Manager**
2. Enter the wallet address
3. Click **Prepare** — Erebor calls `POST /api/v1/users/secondary-manager/prepare` and returns calldata
4. Sign the on-chain transaction
5. The new secondary manager appears after on-chain confirmation

### Removing a Secondary Manager

1. Click **Remove** next to the manager
2. Click **Prepare** — Erebor calls `POST /api/v1/users/secondary-manager/remove/prepare`
3. Sign the on-chain transaction

### Updating Display Names

Click the pencil icon next to any user to update their display name. This is an off-chain operation (no gas required):

```bash
PATCH /api/v1/users/{wallet_address}/display-name
{ "display_name": "Alice (Operations)" }
```

## Tab 3: View Only

View-only users can see the Dashboard but cannot make any changes. No on-chain transaction required.

### Adding a View-Only User

```bash
POST /api/v1/users/view-only
{
  "vault_address": "0x...",
  "chain_id": 8453,
  "wallet_address": "0xViewer...",
  "display_name": "Risk Monitor"
}
```

### Removing a View-Only User

```bash
DELETE /api/v1/users/view-only/{wallet_address}?vault_address={}&chain_id={}
```

## API Reference

| Endpoint | Method | Auth | Description |
|----------|--------|------|-------------|
| `/api/v1/users` | GET | JWT | All users for vault |
| `/api/v1/users/secondary-manager/prepare` | POST | JWT | Prepare add tx (returns calldata) |
| `/api/v1/users/secondary-manager/remove/prepare` | POST | JWT | Prepare remove tx |
| `/api/v1/users/view-only` | POST | JWT | Add view-only (off-chain) |
| `/api/v1/users/view-only/{wallet}` | DELETE | JWT | Remove view-only (off-chain) |
| `/api/v1/users/{wallet}/display-name` | PATCH | JWT | Update display name |

```json
    }

```

<Tip>
Add at least one secondary manager as a backup before going live. If you lose access to the primary manager key, secondary managers can continue vault operations.
</Tip>
